# 07 - Backend API spec

Base URL for development:

```text
http://localhost:3000/api
```

Production/staging must use HTTPS.

## Common headers

```http
Authorization: Bearer <access_token>
Content-Type: application/json
X-Device-Id: <device_id>
X-Idempotency-Key: <uuid>   # for create/submit actions when needed
```

## Common response format

Success:

```json
{
  "data": {},
  "message": "OK"
}
```

Error:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Dữ liệu không hợp lệ",
    "details": []
  }
}
```

```json

  "password": "password"
}
```

Response:

```json
{
  "accessToken": "...",
  "refreshToken": "...",
  "user": {
    "id": "...",
    "fullName": "Nhân viên 1",
    "role": "STAFF",

```

### POST /auth/refresh

Request:

```json
{
  "refreshToken": "..."

```

### POST /auth/logout

Revoke refresh token.

### GET /auth/me

Return current user profile.

## Purchase receipt endpoints

### GET /purchase-receipts

Allowed:

```text
STAFF: own receipts only
MANAGER: all farm receipts
ADMIN: all farm receipts
```

Query:

```text
?status=SUBMITTED
?from=2026-06-01
?to=2026-06-30
```

### GET /purchase-receipts/:id

Return receipt detail with items.

### POST /purchase-reipts

Typo warning: the actual endpoint should be `/purchase-receipts`.

Allowed:

```text
ADMIN, MANAGER, STAFF
```

Required header:

```http
X-Idempotency-Key: <uuid>
```

Request:

```json
{
  "clientRequestId": "uuid-from-mobile",
  "receiptDate": "2026-06-01",
  "supplierName": "Đại lý vật tư A",
  "note": "Nhập đầu vụ",
  "items": [
    {
      "materialId": "optional-material-id",
      "materialName": "Thức ăn CP 9001",
      "quantity": 10,
      "unit": "bao",
      "unitPrice": 450000
    }
  ]
}
```

Backend calculates:

```text
line_total = quantity * unit_price
total_amount = sum(line_total)
receipt_code = generated by server
status = DRAFT
```

### PATCH /purchase-receipts/:id

Allowed:

```text
Creator can edit own DRAFT receipt
MANAGER/ADMIN can edit DRAFT receipt in farm
```

Do not allow editing submitted/approved receipts.

### PATCH /purchase-receipts/:id/submit

Allowed:

```text
Creator, MANAGER, ADMIN
```

Rules:

- Receipt must be DRAFT or REJECTED if resubmission is allowed
- Must have at least 1 valid item
- Change status to SUBMITTED
- Set submitted_by and submitted_at
- Write audit log

### PATCH /purchase-receipts/:id/approve

Allowed:

```text
MANAGER, ADMIN
```

Rules:

- Receipt must be SUBMITTED
- Run database transaction
- Create inventory transactions
- Update inventory balances
- Set status APPROVED
- Write audit log

### PATCH /purchase-receipts/:id/reject

Allowed:

```text
MANAGER, ADMIN
```

Request:

```json
{
  "reason": "Sai số lượng"
}
```

### PATCH /purchase-receipts/:id/void

Allowed:

```text
MANAGER, ADMIN
```

Request:

```json
{
  "reason": "Phiếu bị nhập nhầm"
}
```

Rules:

- If receipt already affected inventory, create reverse inventory transactions
- Set status VOIDED
- Write audit log

## User management endpoints

Phase 1 minimal admin only:

```text
GET /users
POST /users
PATCH /users/:id
PATCH /users/:id/disable
PATCH /users/:id/password
```

Admin creates:

- 1 manager
- 4 staff

Notes:

- No public self-registration in the staff APK.
- Admin sets an initial password or resets a password from web.
- Passwords are hashed before storage.
